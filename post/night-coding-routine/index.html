#生活

深夜编程的仪式感

2026-05-08 1 分钟阅读 272 字 阅读 0 次

21:30 清场

把白天的消息、通知、TODO 清掉。浏览器标签从二十几个关到只剩三个：文档、GitHub、音乐。

这不是效率技巧，是心理上的"进入状态"。

22:00 音乐

深夜编程一定要有 BGM。我的歌单分三类：

  • 重构/写测试 → Lo-fi Hip Hop
  • 学新东西/看文档 → 久石让的纯音乐
  • 修 bug → 无音

选错的后果：听着 Radiohead 看文档，五分钟后就不知道自己在看什么了。

22:30 真正开始

前半小时是预热的。经常写了一段然后删掉重来，这是正常的。

我学会了不在这段时间给自己压力。

00:00 中场

站起来，去厨房倒水。看窗外几秒。这个停顿很重要——经常回来后发现刚才的逻辑有问题。

01:30 收工

不管做没做完，凌晨一点半收工。半夜写的代码第二天看大概率要改，不如早点睡，明天思路更清晰。

这套流程不是为了装酷，是发现规律后的总结。你的深夜习惯是什么？
